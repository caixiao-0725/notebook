# 1. git stash

`git stash` 是一个非常有用的 Git 命令，它的主要作用是**临时保存当前工作目录的修改**，让你可以切换分支或执行其他操作，之后再把保存的修改重新应用回来。

### 核心用途

1. **临时切换分支**
    
    - 当你在一个分支上工作到一半，突然需要切换到其他分支处理紧急任务时
        
    - 但又不想提交半成品代码（可能无法通过测试或不符合提交规范）
        
2. **清理工作目录**
    
    - 让你的工作目录暂时回到干净的状态（与最后一次提交一致）
        
    - 便于执行 `git pull`、`git merge` 等操作
        
3. **保存实验性修改**
    
    - 尝试一些代码改动，但不确定是否采用，可以先 stash 起来
        

### 常用命令

```powershell
# 保存当前修改（包括暂存区和未暂存的）
git stash
# 保存并添加说明信息
git stash save "说明文字"
# 查看所有 stash 列表
git stash list
# 应用最近一次的 stash（但不删除记录）
git stash apply
# 应用最近一次的 stash 并删除记录
git stash pop
# 删除最近一次的 stash
git stash drop
# 清空所有 stash
git stash clear
```

总之，`git stash` 就像是一个**临时草稿箱**，让你可以随时保存工作进度，处理完其他事情后再回来继续。
