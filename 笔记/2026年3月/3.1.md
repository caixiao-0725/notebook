# 1. 刚体加速

## 1.gpu内存

cuda graph操作的内存地址不能变化，不能随意 cuda malloc 和 free 。

CUDA Graph在录制时**固化（冻结）了所有指针的内存地址**。当你：

1. 录制时：`qPartALocal` 指向地址 `0x1234`
    
2. 释放后重新malloc：`qPartALocal` 指向新地址 `0x5678`
    
3. 执行图时：图仍然尝试访问原来的地址 `0x1234`
## 2.cpu内存

当你这样写核函数时：
```cpp
kernel<<<...>>>(qPartALocal, a);  // a 是 CPU 上的 int 变量
```

CUDA Graph 录制时，这个 `int a` 的**值**会被固化到图中。如果你后续修改了 `a` 的值，图执行时**仍然使用原来的值**！
